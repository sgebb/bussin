Security & Permissions: How Bussin Handles Your Data

A detailed look at the zero-backend architecture, consent flows, and why your credentials never leave your browser.

The Short Answer

Your credentials never touch Bussin's infrastructure. There is no Bussin server. When you use Bussin, your browser talks directly to Microsoft and Azure.

Zero-Backend Architecture

Bussin is a Blazor WebAssembly application - it compiles to JavaScript and runs entirely inside your browser. There's no API server, no message proxy, no database. The app is just static files served from a CDN.

This means:

  • No Bussin server can log your messages
  • No credentials are stored server-side
  • There is no account system - Bussin doesn't know who you are

How Authentication Works

Bussin uses MSAL.js (Microsoft Authentication Library) to authenticate you via Entra ID. Here's the flow:

  1. You click Sign in with Microsoft
  2. Your browser redirects to login.microsoftonline.com - Microsoft's servers
  3. You authenticate with your Microsoft account (MFA applies as normal)
  4. Microsoft returns an OAuth 2.0 access token to your browser
  5. That token is stored in your browser's session storage only

Bussin receives the token but never transmits it to any external server. It uses it to sign API calls sent directly to Azure.

What Permissions Are Requested?

Bussin requests two OAuth scopes:

  • https://servicebus.azure.net/user_impersonation - the Azure Service Bus Data Plane API. This lets Bussin peek, send, and manage messages in namespaces you already have access to.
  • https://management.azure.com/user_impersonation - the Azure Management API. This lets Bussin discover your namespaces automatically, so you don't have to type connection strings.

Both use your existing Azure RBAC permissions. Bussin cannot access any resource you wouldn't already have access to in the Azure Portal.

AMQP over WebSockets

After authentication, message operations use the Rhea AMQP library to open a secure WebSocket connection to your Service Bus namespace. The connection goes from your browser directly to <namespace>.servicebus.windows.net - no intermediary.

Revoking Access

You can revoke Bussin's consent at any time from myapplications.microsoft.com under Apps & Services.

(Info) Open Source
You don't have to take our word for it. Bussin is fully open source. You can inspect the authentication code and message handling on GitHub.
