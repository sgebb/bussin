Scheduled Messages

Service Bus supports scheduled enqueue â€” the message is accepted immediately but only becomes visible to consumers at the scheduled time. Bussin offers two ways to schedule:

Absolute datetime

Use the Scheduled Enqueue (UTC) datetime picker to select an exact date and time. Bussin validates that the time is in the future.

Relative time

Use the Relative Schedule field for quick scheduling. Type a duration like:

  • 30s â€” 30 seconds from now
  • 5m â€” 5 minutes from now
  • 2h â€” 2 hours from now
  • 1d â€” 1 day from now

The relative time takes precedence over the absolute datetime if both are specified.

Custom Application Properties

Expand the Custom Properties accordion to add key-value pairs that your consumer can read. Each property has a type selector:

  • String â€” plain text
  • Number â€” parsed as a numeric value
  • Boolean â€” true or false

Add as many properties as needed. These are perfect for routing metadata, versioning, or debugging context.

Batch Send from JSON

Switch to the Batch Send tab to upload a JSON file. The file must be a JSON array â€” each element becomes a separate message body. Bussin shows a preview of the first 10 messages and a count of the total.

Example file:

["Order 1", {"orderId": 123, "status": "pending"}, "Order 3"]

Files up to 5 MB are supported. This is invaluable for replaying test scenarios or seeding a queue with sample data.
