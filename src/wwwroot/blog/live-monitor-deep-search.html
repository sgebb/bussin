Live Monitor Mode

Live Monitor lets you watch messages arrive on a queue or subscription in real-time. It uses peek mode â€” messages are not consumed or deleted. Think of it as tail -f for your Service Bus.

  3. Check Live Monitor Mode

Messages are deduplicated by sequence number, so you won't see duplicates even if the polling catches the same message twice. Click Stop Monitoring to pause, or close the modal entirely.

Deep Search

Standard peek is limited to 10,000 messages. But what if you have 500,000 messages and need to find a specific one? That's where Deep Search comes in.

How it works

When you add a search filter (Message ID, Subject, or Body content) in Advanced Peek and the entity has more than 10,000 messages, Bussin automatically suggests Deep Search Mode. This launches the search as a background task that scans up to 1 million messages.

  1. Open Advanced Peek
  2. Enter a filter â€” for example, a MessageId or a text snippet in the body
  3. If the entity has >10K messages, you'll see the Deep Search option
  4. Click Start Deep Search

Tracking progress

The search runs in the background. You can:

  • See real-time progress in the Running Tasks panel (bottom-left)
  • Continue using other parts of the app â€” peek other queues, send messages, etc.
  • Cancel the search at any time
  • View results when it completes â€” Bussin shows the matching sequence numbers and lets you jump to those messages

Filter options

All three filters use case-insensitive partial matching:

  • Message ID â€” search by the broker-assigned or custom message ID
  • Subject â€” search the Subject/Label property
  • Body Content â€” search inside the message body text
