Inside Bussin: Architecture & Open Source

How a Blazor WASM + Rhea AMQP app talks directly to Azure from the browser. With links to the GitHub source.

The Core Constraint: Browsers Can't Open TCP Sockets

Azure Service Bus uses AMQP 1.0 over TCP â€” a binary protocol that browsers can't speak natively. The classic desktop Service Bus Explorer solves this by running as a native Windows process. Bussin takes a different approach: AMQP over WebSockets.

Azure Service Bus exposes a WebSocket endpoint that acts as a tunnel for AMQP frames. Browsers are very capable of WebSocket connections, which means a properly tunneled AMQP client can run entirely in a browser.
